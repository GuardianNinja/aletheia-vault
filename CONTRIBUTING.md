# Contributing to Aletheia Vault

Thank you for your interest in contributing.

---

## Ways to contribute

- **Architecture and design**
  - Review and refine the documents in `docs/`.
- **Backend development**
  - Implement services for auth, inventions, licensing, payouts, and markets.
- **Frontend development**
  - Build the web UI for inventors, organizations, and the public.
- **Security and governance**
  - Help define and audit security controls and policies.
- **Legal and policy**
  - Suggest improvements to licensing frameworks and governance models.

---

## Contribution process

1. **Open an issue**
   - Describe what you want to add or change.
2. **Discuss**
   - Align with maintainers and other contributors.
3. **Fork and branch**
   - Create a feature branch in your fork.
4. **Submit a pull request**
   - Include a clear description and references to related issues.
5. **Review**
   - Maintain a respectful and collaborative tone.

---

## Code of conduct

Contributors are expected to uphold a standard of respect, inclusion,
and constructive feedback. Harassment, discrimination, and abuse are
not tolerated.

A full code of conduct may be added in a later iteration.
